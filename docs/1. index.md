# Introduction

**HuskHoard** is a Hybrid User-Space Storage Kernel designed to solve the "NVMe Capacity Gap." It provides transparent, automated data tiering for Linux, allowing you to treat slow, high-capacity storage (**HDDs, Physical LTO Tape, Cloud Buckets**) as a seamless extension of your high-speed SSD.

## The Problem
Modern storage is polarized. NVMe SSDs are fast but small. Enterprise storage (LTO Tape, S3, Cold HDDs) is cheap and massive but requires specialized tools and manual management.

## The HuskHoard Solution
HuskHoard automates this process using **Transparent Stubbing** and **StreamGate**:
1.  **The Husk:** Moves cold data to a storage pool and "punches a hole" in the SSD file.
2.  **The Recall:** Intercepts reads and rehydrates files in real-time.
3.  **StreamGate:** Allows you to stream massive files (videos, logs) directly from tape/cloud to an application **without** ever writing them back to your SSD.

## Our Philosophy
*   **Sequential-First:** We treat all archive media like Tape. By writing sequentially, we eliminate the "write-wall" of SMR drives and natively support LTO-5 through LTO-9 hardware.
*   **Zero-Disk Footprint:** With StreamGate, your SSD capacity no longer limits the size of the files you can "watch" or "process."
*   **Hardware Native:** Direct SCSI `ioctl` support for physical tape drives and `O_DIRECT` for block devices.
