### Section 5: User Guides (`usage.md`)

# User Guides

Once the HuskHoard daemon is running, it operates largely in the background. However, you have full control over the archival policies and recovery process.

## 1. Controlling Archival Policies
Archive behavior is defined in `husk_config.toml`.

### Age-Based Archiving
The most common use case is archiving files once they haven't been touched for a certain period.
```toml
max_age_days = 30 # Files not accessed in 30 days become Husks
```

### Immediate Archiving
Some files (like raw video footage or ISOs) should never take up SSD space. You can force these to archive as soon as the Janitor sees them:
```toml
immediate_archive_extensions = ["mp4", "mkv", "iso", "zip"]
immediate_archive_dirs = ["/ArchiveDrop/"]
```

## 2. Restoring Data
HuskHoard provides two ways to get your data back.

### A. Transparent Rehydration (The "Just Open It" Method)
Simply open the file in any application (VLC, Image Viewer, Text Editor). HuskHoard will intercept the read, fetch the data from the Hoard, and fill the "Hole" on your SSD. The application will continue as if the file was always there.

### B. Manual CLI Restore (PITR)
HuskHoard maintains a history of your files. If you accidentally overwrite a file and want the version from three days ago, use the `restore` command:
```bash
./huskhoard restore --file-path /home/jm/report.pdf --dest-path ./recovered.pdf --version 1
```

## 3. Monitoring Volume Health
Use the `info` command to see the "Tank Gauge" of your storage volumes.
```bash
./huskhoard info --tape-dev /dev/sdb
```
This will display:
*   **Physical Usage:** How much of the drive is physically written.
*   **Active Data:** How much of that data represents current files.
*   **Reclaimable Space:** Space occupied by old versions or deleted files that can be recovered by running a **Repack**.

## 4. Understanding Log Levels
If you are troubleshooting, change the `log_level` in your config:
*   `error`: Only show critical failures.
*   `info`: Default. Shows archival and restore activity.
*   `debug`: Shows detailed fanotify events and database queries.
