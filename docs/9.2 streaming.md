# HTTP Streaming Gateway

The **HuskHoard Streaming Gateway** is designed to solve the problem of "Massive Media on Slow Hardware." Instead of waiting for a 80GB 4K Blu-ray rip to rehydrate from tape to your SSD, you can stream it directly into your player.

## 1. How it Works
The gateway acts as a "Translator." 
1. **The Player** (VLC/Plex) asks for: `GET /stream/movie.mkv` with a header `Range: bytes=500MB-600MB`.
2. **HuskHoard** looks up the Jump-Table in the Catalog.
3. **StreamGate** identifies the exact 16MB Zstd frames on the Physical Tape or Cloud.
4. **The Engine** extracts only those bytes, decodes them in RAM, and pipes them directly into the HTTP Socket.

**Result:** You get O(1) seeking (instant jumps) on files stored on linear media like LTO Tape or S3.

## 2. Configuration
In your `husk_config.toml`, you can set the port:
```toml
http_port = 8080
```
By default, the gateway binds to `127.0.0.1`. This is a security feature to ensure your archive isn't exposed to the internet. If you need to access it from another machine, we recommend using an Nginx reverse proxy or a VPN (Tailscale).

## 3. Media Server Integration

### VLC / MPV
To watch a file instantly, simply point your player at the HuskHoard URL:
```bash
vlc http://127.0.0.1:8080/stream/my_movie.mp4
```

### Plex / Jellyfin
To use HuskHoard as a backend for a media server:
1. Create a `.strm` file (e.g., `Gladiator.strm`).
2. Inside that file, paste the HuskHoard URL: `http://127.0.0.1:8080/stream/Movies/Gladiator.mkv`.
3. Add the folder containing the `.strm` files to your Plex/Jellyfin library.

Plex will now treat the LTO Tape or S3 bucket as a standard high-speed stream.

## 4. Performance & Hardware Safety
*   **Zero-Disk Footprint:** Data flows from the archive media -> RAM -> Socket. The "Stub" file on your SSD is never overwritten during streaming, preserving your SSD's lifespan and capacity.
*   **Shoe-Shining Prevention:** When streaming from physical LTO tape, HuskHoard uses its 256KB buffer. If the player pauses the video, the gateway applies "Backpressure," pausing the tape hardware cleanly to prevent head wear.
*   **Broken Pipe Handling:** Media players frequently drop connections when seeking. HuskHoard gracefully kills the extraction thread and starts a new one at the new offset, preventing "Database Locked" errors or daemon crashes.

## 5. Security & Stability
*   **Path Traversal Protection:** The gateway strictly validates URLs. You cannot use `../` to escape the `hot_tier` and read system files like `/etc/shadow`.
*   **Localhost Enforcement:** The gateway is designed for local-to-local communication. This keeps your metadata and file content off the open web.
*   **Graceful 503s:** If you try to stream a file from a tape that is currently sitting on your shelf (offline), the gateway will return an `HTTP 503 Service Unavailable` with a log entry telling you exactly which drive to plug in.
