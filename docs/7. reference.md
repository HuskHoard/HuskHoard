### Section 7: Technical Reference (`reference.md`)

# Technical Reference

This section provides the low-level specifications for the HuskHoard storage format and CLI.

## 1. The USTD Object Header (4096 bytes)
Every object on a HuskHoard volume begins with a 4KB header, aligned to the disk's physical sectors.

| Offset | Size | Field | Description |
| :--- | :--- | :--- | :--- |
| 0 | 8B | `magic_bytes` | `USTD\x00\x01\x02\x03` |
| 8 | 2B | `format_version` | Current version: 1 |
| 10 | 2B | `flags` | Operational flags (e.g., deleted, partial) |
| 12 | 1B | `compression_type` | 0=None, 1=Zstd |
| 13 | 3B | `reserved_pad` | Structural padding to maintain 64-bit alignment |
| 16 | 8B | `payload_size` | Uncompressed size in bytes |
| 24 | 8B | `compressed_size` | Compressed size in bytes |
| 32 | 8B | `padded_size` | Total size aligned to 4KB sectors |
| 40 | 16B | `object_uuid` | Unique ID for this specific version |
| 56 | 16B | `tape_uuid` | ID of the volume this object belongs to |
| 72 | 8B | `mtime` | Original POSIX modified time |
| 80 | 8B | `ctime` | Original POSIX created time |
| 88 | 4B | `posix_mode` | Original read/write/execute permissions |
| 92 | 4B | `uid` | Original POSIX User ID |
| 96 | 4B | `gid` | Original POSIX Group ID |
| 100 | 32B | `data_checksum` | BLAKE3 cryptographic hash of payload |
| 132 | 4B | `header_crc32` | CRC32 integrity check for bytes 0-131 |
| 136 | 3960B| `tlv_data` | Type-Length-Value packed variable metadata |

### TLV Metadata Packing
The `tlv_data` block stores variable-length information:
*   **Type 0x01:** Original Filename.
*   **Type 0x02:** Linux Extended Attributes (xattrs).

## 2. CLI Command Summary

| Command | Argument | Description |
| :--- | :--- | :--- |
| `daemon` | None | Starts the interceptor and policy engine. |
| `format` | `--tape-dev` | Writes a Volume Header to LBA 0 of a drive. |
| `info` | `--tape-dev` | Displays capacity and "Wasteland" statistics. |
| `scrub` | `--tape-dev` | Verifies BLAKE3 hashes for all data on volume. |
| `restore` | `--file-path` | Manually extracts a file (supports `--version`). |
| `repack` | `--source`, `--dest` | Reclaims space by migrating active data. |
| `rebuild` | `--tape-dev` | Recovers a lost database by scanning hardware. |

## Storage Interface Specs
*   **Block Size:** 4096 bytes (Logical), 262,144 bytes (Physical SCSI Tape Frame).
*   **Tape Markers:** Uses standard SCSI `EOF` filemarks between every archived object.
*   **Seek Method:** `lseek()` for disks, `MTFSF` (Forward Space File) for physical tape.

## 3. Performance Tuning
*   **Alignment:** HuskHoard enforces 4096-byte alignment. This ensures compatibility with **4Kn Advanced Format** drives and prevents "Partial Sector Writes."
*   **O_DIRECT:** HuskHoard uses `O_DIRECT` to bypass the Linux Page Cache. This prevents archival tasks from "starving" your system RAM, keeping your desktop responsive while large backups occur.
*   **Zstd Window:** HuskHoard uses a sliding Zstd window optimized for 16MB frames, balancing high compression ratios with fast random-access rehydration.
